反转链表有几种转法？

0. Preface

开头先说几句记录心情的废话，考研结束后去牛客看了一下各位大佬的面经，感觉恍如隔世。对于一个标准的Java Boy而言，当看到一篇实习岗位的八股文竟然要涉及到JVM底层调优时，不由得一阵心悸。没想到Java的开发岗已经如此之卷，这也坚定了我要写本篇博文的信心。既然要卷，做不了Java Boy们的卷王，那就做Java Boy们的”孔乙己“吧。

本篇依然是”茴香豆“系列，讲述一下反转链表的”转“有几种”写法“。

1. 反转链表

反转链表这道题在Leetcode上被定级为简单题，在众多的链表题型中，反转操作确实没有什么排面。不过在一些公司的笔试中出现的频率也很高，算得上是算法八股的常客了。

LeetCode题目链接：https://leetcode.cn/problems/reverse-linked-list/

题目描述：给你一个单向链表，请在线性时间复杂度内，完成对链表结点的反转。注意：不要更改链表结点的数据，只更改链表结点的指针域。

2. 三指针

三指针的做法是最为经典的一种实现方式，如上图所示，初始时准备三个指针，分别为prev指向前驱结点，ptr指向当前结点，next指向后继结点。next指针的作用在于保存后继结点的引用，因为在翻转的过程中，要修改ptr的next域，若不事先缓存，就会造成后继结点信息的丢失。

每次翻转时，只在局部翻转，即当前的ptr.next指向prev，随后，三个指针同时后撤：prev <- ptr, ptr <- next, next <- ptr.next。

3. 递归

递归的做法，是三种实现中最”妙“的一种实现，采用三指针的方式，需要在代码编写时，准备多个辅助指针，完成相关结点的记录和缓存。但是若我们能够从链表尾部（链表的最后一个节点）出发，只需作出如下指针变更：head.next.next <- head即可完成反转操作。使用递归的方式，实质上是利用函数调用堆栈，保存每个结点的信息。

4. 双指针

双指针的做法与三指针无异，因为head结点在返回前，都没有用处，因此，可以使用头指针指向的head结点来替代三指针中的next引用，保存后继节点信息，因此双指针和三指针在思想上是差不多的。
